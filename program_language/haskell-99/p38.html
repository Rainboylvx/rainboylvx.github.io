P38 - 高欧拉函数数

Construct the list of highly totient numbers

官方模块：Problems.P38　核心函数：highlyTotientNumbers

← P37 欧拉乘积公式 | P39 素数列表 →

题目描述

可能有多个 xxx 满足 φ(x)=n\varphi(x)=nφ(x)=n。若 nnn 的原像数量严格大于所有更小整数的原像数量，就称 nnn 为高欧拉函数数（highly totient number）。要求构造它们的无限列表。

思路

困难在于不能无限枚举 xxx。若 n=φ(x)n=\varphi(x)n=φ(x)，可由 nnn 的质因数构造一个安全上界

x≤∏q∈{1}∪factors⁡(n)(q+1). x\le \prod_{q\in\{1\}\cup\operatorname{factors}(n)}(q+1). x≤q∈{1}∪factors(n)∏​(q+1).

因此 tally n 只需检查有限区间。随后从 1 起扫描，只输出打破历史最大原像数的 nnn。

实现

 1highlyTotientNumbers :: [Integer]
 2highlyTotientNumbers = records 1 0
 3  where
 4    records n best
 5      | count > best = n : records (n + 1) count
 6      | otherwise    = records (n + 1) best
 7      where count = tally n
 8
 9tally :: Integer -> Int
10tally n = length [x | x <- [1..upperBound n], totient' x == n]
11
12upperBound :: Integer -> Integer
13upperBound n = product (map (+ 1) (1 : primeFactors n))

这里复用了 P35 的 primeFactors 和 P37 的 totient'。这是为表达题意设计的实现，并不适合计算很靠后的项。

测试

1>>> take 10 highlyTotientNumbers
2[1,2,4,8,12,24,48,72,144,240]

要点总结

  • 无限结果不意味着每一步也必须进行无限搜索。
  • 先证明有限上界，再用惰性列表逐项输出，是这题的核心。
