P23 - 从列表随机选择 N 个元素

Extract a given number of randomly selected elements from a list

官方模块：Problems.P23　核心函数：randomSelect

← P22 生成整数范围 | P24 随机抽取不同整数 →

题目描述

从列表中随机选出 nnn 个不同位置的元素。一个元素一旦选中就从候选列表删除，因此不会重复抽中同一位置。函数还要返回更新后的随机数生成器。

函数签名

1import System.Random (RandomGen)
2
3randomSelect :: RandomGen g => [a] -> Int -> g -> ([a], g)

显式传入和返回生成器，能让函数保持纯函数：同一列表、数量和种子总会产生同一结果。

实现

每轮生成一个合法下标，用 removeAt0 同时取出元素和剩余列表，然后继续抽取。

 1import System.Random (RandomGen, randomR)
 2
 3randomSelect :: RandomGen g => [a] -> Int -> g -> ([a], g)
 4randomSelect xs count gen = go xs (max 0 count) gen []
 5  where
 6    go [] _ g picked = (reverse picked, g)
 7    go _  0 g picked = (reverse picked, g)
 8    go pool k g picked =
 9      let (i, g')       = randomR (0, length pool - 1) g
10          (x, pool')    = removeAt0 i pool
11      in go pool' (k - 1) g' (x : picked)
12
13removeAt0 :: Int -> [a] -> (a, [a])
14removeAt0 i xs =
15  case splitAt i xs of
16    (front, x : back) -> (x, front ++ back)
17    _ -> error "removeAt0: index out of bounds"

这里的错误分支不会从 randomSelect 触发，因为 randomR 的范围由当前列表长度决定。

测试

1import System.Random (mkStdGen)
2
3>>> length . fst $ randomSelect "abcdefgh" 3 (mkStdGen 111)
43
5>>> let (picked, _) = randomSelect [1..5] 10 (mkStdGen 1)
6>>> length picked == 5
7True
随机测试

不要把某个版本 random 包产生的具体排列当成唯一正确答案。更稳定的性质是：结果长度正确、每个结果都来自输入，并且同一位置不会抽取两次。

复杂度

列表的 length、splitAt 和 ++ 都是线性操作。抽取 kkk 次的最坏复杂度为 O(kn)O(kn)O(kn)；若追求性能，应换用可随机访问并高效删除的数据结构。
